8. 路线图

产品阶段性规划与里程碑。详细版本见 roadmaps/ROADMAP.md。

Phase 0: 项目骨架 + 通信验证 ✅ 完成

2026-07-03

  • Go 模块初始化 + 目录结构
  • WebSocket Hub（gorilla/websocket）
  • AgentEvent 系统（18 种事件类型）
  • SQLite 6 张表 Schema
  • Vue 3 CDN 前端 + 事件路由器
  • /api/tasks stream-demo 端到端测试
  • OpenSpec 全部产物（proposal, design, 6 specs, tasks）

Phase 1: Agent Loop 核心引擎 ✅ 完成

2026-07-03

  • OpenAI-compatible LLM Client（内部 API + SSE streaming）
  • 3 个内置工具实现（run_shell, write_file, read_file）
  • ReAct Loop 引擎（think → tool_call → observe → loop）
  • Step 状态机 + 事件发送
  • Agent 配置加载（SQLite → LLM 绑定）
  • DB 持久化 + Agent CRUD API + Task 历史查询
  • 安全加固 + 白盒注释铁律

Phase 2: 前端可视化 ✅ 完成

2026-07-03

  • Vue 3 Vite 迁移 + TypeScript
  • AgentTree 组件（递归树 + 实时更新）
  • TypeWriter 组件（LLMDelta 流式渲染 + marked + highlight.js）
  • Markdown 实时渲染 + 语法高亮
  • Step 展开/折叠 + StatusIndicator 状态指示器
  • Pause / Resume / Cancel 控制 + 指标面板
  • Go embed 集成（前端 dist/ 嵌入二进制，单文件部署）
  • 独立部署支持（Vite dev server 代理到 Go 后端）

Phase 3: 预设 Cases + 配置页面 + Harness 基础 ✅ 完成

2026-07-03

  • 5 个预设 Task Cases（代码生成、研究、多Agent、对话、长任务）
  • CaseCard UI 组件 + 一键执行
  • Harness: TaskContract 定义 + Progress 文件管理
  • Harness: FileScopeRule + PathTraversalRule（路径安全）
  • Harness: AcceptanceCriteria 基础实现
  • Harness: PolicyGate 集成到 Engine（executeTool 拦截）
  • /api/cases 端点 + ?case= 参数支持
  • Agent CRUD 前端页面 + 任务历史侧边栏 → Phase 4
  • Memory: Task 摘要 → Phase 4

Phase 4: 多 Agent 并发 + Harness 控制层 + 记忆基础 ✅ 完成

  • 多 Agent Task 分派（goroutine 并行）
  • 前端多树渲染（并排或选项卡，颜色区分）
  • Harness: PolicyChain 完整实现（PolicyGate + 内置规则链）
  • Harness: TokenBudgetRule + ToolWhitelistRule
  • AgentBus 通信层代码落地
  • 前端体验优化：全局版本、loading、Toast、智能滚动、详情弹窗、快捷键、token 细节、max steps 配置
  • Agent 间通信协议 → Phase 5
  • ModelProfile + ModelRegistry → Phase 5
  • Checkpoint / Recovery → Phase 5
  • Memory: memories 表 + 心跳整理器 → Phase 5

Phase 5: 运行时注册 + Provider + Router + 记忆召回 + 会话历史 🔜 当前

  • 运行时 Tool 注册 REST API
  • AI 自描述工具注册
  • Docker 沙箱（run_shell 安全隔离）
  • LLM Provider 接口抽象 + OpenAIProvider 基线
  • Router 路由决策（意图分类 + 模型选择）
  • Harness: ApprovalRule（高风险操作前端确认）
  • Harness: DangerousCommandRule（Shell 命令危险模式检测）
  • Memory: MemoryRecall 召回 + Working Memory 构建
  • Memory: 记忆冲突检测 + 合并
  • 会话与任务历史（新增）：新建会话、历史会话切换、继续执行/查阅

Phase 6: 高级特性 + 多厂商 + 成本控制 + 记忆治理 远期

  • RAG 向量检索（Embedding + 向量数据库）
  • 用户认证 / 多租户
  • gRPC 协议迁移
  • 可观测性（OpenTelemetry + Prometheus）
  • Prompt Template 引擎
  • 多厂商 Provider（Anthropic + DeepSeek reasoning）
  • Worker Pool 并发调度 + CostTracker 成本追踪
  • 降级策略（Opus→Sonnet→DeepSeek→本地）
  • Harness: Eval 回归闭环（失败 Trace → 回归测试集）
  • Harness: 完整治理与审计（身份 + 审批 + 成本）
  • Memory: 向量检索增强（LanceDB / ChromaDB）
  • Memory: 遗忘曲线 + 冷存储 + 记忆审查 Agent
