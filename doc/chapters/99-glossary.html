术语表

术语说明
AgentAI 代理，绑定特定 LLM 配置、System Prompt、工具集，可独立执行任务的智能体
ReAct LoopReason + Act 循环：Agent 思考(think) → 调用工具(tool_call) → 观察结果(observation) → 继续思考的迭代过程
StepAgent 执行的一次循环单位，类型为 think / tool_call / observation
Task用户发起的一次任务，可包含 1 个或多个 Agent，有完整的生命周期(running→completed/failed)
ToolAgent 可调用的外部能力（如 shell 执行、文件读写），通过 JSON Schema 描述参数
Tool Registry工具注册中心，管理所有可用工具的注册、查询、执行
Event系统内部通信的最小单位，通过 WebSocket 在前后端之间传输
EventType事件类型枚举，共 18 种，覆盖生命周期 / LLM / 工具 / 控制 / 系统事件
HubWebSocket 连接管理中心，管理所有客户端连接、注册、注销、广播
LLM DeltaLLM 流式输出中的增量文本，前端累积后实时渲染
SSEServer-Sent Events，HTTP 单向流协议，用于 LLM token 流式传输
WebSocket全双工通信协议，本平台的前后端实时通信基础
gRPCGoogle RPC 框架，Phase 6+ 的协议迁移目标，提供类型安全和双向流
OpenAI-compatible与 OpenAI API 格式兼容的接口标准，内部 API 遵循此标准
System Prompt系统提示词，定义 Agent 的角色、行为准则和约束
Function CallingLLM 输出结构化 JSON 来触发工具调用的能力
TokenLLM 处理文本的最小语义单元，用于计费和量化上下文长度
Case Card前端预设任务卡片，用户可一键发起预设的 Agent 任务
Agent Tree前端 Agent 状态可视化组件，以树形结构展示 Agent 的每一步执行过程
TypeWriter前端流式文本渲染效果，模拟打字机逐字显示 LLM 输出
RAGRetrieval-Augmented Generation，检索增强生成，Phase 6+ 的知识库能力
OpenSpec项目采用的变更管理框架，包含 proposal → design → specs → tasks 工件流
Phase开发阶段，每个 Phase 有独立的目标和验证标准，完成后提交 Git
