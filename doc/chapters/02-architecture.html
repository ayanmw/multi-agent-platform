2.3 数据流

用户发起任务
    │
    ▼
前端 POST /api/tasks { action, input }
    │
    ▼
后端创建 Task + Agent 实例
    │
    ▼
ReAct Loop 开始
    │
    ├─► 调用 LLM API（SSE streaming）
    │       │
    │       ├─► llm_thinking 事件 ──► WebSocket ──► 前端实时渲染
    │       ├─► llm_delta 事件   ──► WebSocket ──► 前端 typewriter
    │       └─► llm_message_complete
    │
    ├─► 解析 Function Call
    │       │
    │       ├─► tool_call_started  ──► 前端高亮工具名称
    │       ├─► tool_call_input    ──► 前端展示参数
    │       ├─► [执行工具]
    │       ├─► tool_call_output   ──► 前端展示结果
    │       └─► tool_call_complete ──► 前端更新状态
    │
    ├─► 将工具结果追加到上下文
    │       │
    │       └─► 回到 LLM 调用（继续 Loop）
    │
    └─► 最终答案
            │
            ├─► task_completed 事件
            ├─► 持久化到 SQLite（task + steps + conversations）
            └─► 前端展示最终结果

2.4 部署拓扑

环境配置说明
开发单机，localhost:8080Go 后端 + Vue dev server（proxy）或直接访问文件
测试单机，局域网 IPGo builds binary，Vue build → dist/ 由 Go serve
生产（未来）Nginx + systemdNginx 反代 /ws 到 Go 端口，serve static files
⚠️ 当前部署方式
Phase 0-4 阶段，前端使用 web/index.html CDN 直接访问，无需 build step。
Phase 2+ 迁移到 Vite 工程化后，需 npm run build 生成 web/dist/。

2.5 关键设计决策回顾

决策选择原因
通信协议WebSocket + JSON前端原生支持、调试直观、单连接承载所有事件
数据库SQLite (modernc.org)纯 Go 无 CGO、单文件、零运维
Agent Loop自研 ReAct Loop学习优先、最大控制权、可注入任意中间事件
LLM 接入OpenAI-compatible API内部 API 已兼容、标准 streaming format
前端状态Vue reactive（Phase 1→Pinia）快速验证优先，后续工程化
工具执行Go 直接执行（Phase 5+ 沙箱）MVP 阶段允许，文档注明风险
