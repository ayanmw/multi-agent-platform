迁移方式启动时 CREATE TABLE IF NOT EXISTS

5.3 表详细定义

5.3.1 agents — Agent 配置表

字段类型说明
idTEXT PKAgent 唯一标识
nameTEXT UNIQUEAgent 名称
descriptionTEXT描述
system_promptTEXT系统提示词
modelTEXT使用模型（如 deepseek-v4-flash）
temperatureREAL温度参数，默认 0.7
max_tokensINT最大 token 数，默认 4096
api_endpointTEXTLLM API 端点
api_keyTEXTAPI Key（明文存储，Phase 6+ 加密）
toolsJSON允许的工具列表 ["run_shell","write_file"]
configJSON扩展配置（预留）
created_atDATETIME创建时间
updated_atDATETIME更新时间

5.3.2 tasks — 任务执行记录表

字段类型说明
idTEXT PK任务 ID
user_inputTEXT用户输入
statusTEXTrunning / completed / failed
agent_idsJSON参与的 Agent ID 数组
final_resultTEXT最终结果
total_tokensINT总 token 消耗
started_atDATETIME开始时间
completed_atDATETIME完成时间

5.3.3 steps — 执行步骤日志表

字段类型说明
idTEXT PK步骤 ID
task_idTEXT FK关联 tasks.id
agent_idTEXT所属 Agent
step_indexINT第几轮 Loop
typeTEXTthink / tool_call / observation
statusTEXTrunning / completed / failed
contentTEXT思考内容或工具输出
tool_nameTEXT工具名称（type=tool_call 时）
tool_inputJSON工具输入参数
tool_outputTEXT工具执行结果
duration_msINT执行耗时（毫秒）
token_usedINT本步消耗 token
created_atDATETIME创建时间

5.3.4 tools — 工具注册表

字段类型说明
nameTEXT PK工具名称（唯一）
descriptionTEXT工具描述
schemaJSONJSON Schema 参数定义
enabledBOOLEAN是否启用
created_atDATETIME注册时间

5.3.5 conversations — 对话历史表

字段类型说明
idTEXT PK消息 ID
task_idTEXT FK关联 tasks.id
roleTEXTuser / assistant / tool / system
contentTEXT消息内容
created_atDATETIME创建时间

5.3.6 files — 文件元数据表

字段类型说明
idTEXT PK文件 ID
task_idTEXT FK关联任务
filenameTEXT文件名
pathTEXT存储路径（相对 storage/）
size_bytesINT文件大小
mime_typeTEXTMIME 类型
metadataJSON扩展元数据
created_atDATETIME创建时间

5.4 文件存储方案

storage/
├── {task_id}/
│   ├── output.go          # write_file 创建的文件
│   ├── research_report.md
│   └── ...
└── .gitkeep

文件按 task_id 分目录，元数据存入 SQLite files 表。

5.5 会话与任务历史（Phase 5）

当前数据库的 tasks 表已经保存了任务历史，足够支持 Phase 5 的会话管理需求：

  • GET /api/tasks → 查询 tasks 表列表，按 started_at 倒序
  • GET /api/tasks?id=xxx → 查询单个任务 + 关联 steps 详情
  • Session 是前端抽象，不单独建表；Session 名称可由 user_input 前 N 字符生成

5.5.1 是否需要新增表？

Phase 5 第一版保持最小化设计：Session 只在内存 / localStorage 中维护，服务端以 Task 为唯一持久化实体。后续如需多端同步或会话分组，再增加 sessions 表。

5.5.2 索引建议

CREATE INDEX IF NOT EXISTS idx_tasks_status ON tasks(status);
CREATE INDEX IF NOT EXISTS idx_tasks_started_at ON tasks(started_at DESC);
CREATE INDEX IF NOT EXISTS idx_steps_task_id ON steps(task_id);
