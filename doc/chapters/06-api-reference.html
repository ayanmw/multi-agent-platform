6.3 WebSocket — /ws

WebSocket 连接升级后，所有事件以 JSON 格式双向传输。

// 连接
const ws = new WebSocket('ws://localhost:8080/ws');

// 接收事件（服务端 → 客户端）
ws.onmessage = (evt) => {
    const event = JSON.parse(evt.data);
    // event 结构见 4.2 节
};

// 发送控制指令（客户端 → 服务端）
ws.send(JSON.stringify({ type: "pause", task_id: "xxx" }));
ws.send(JSON.stringify({ type: "cancel", task_id: "xxx" }));

6.4 POST /api/tasks — 创建任务

当前请求体已支持 max_steps、timeout_seconds、agent_id、session_id、project_id 等字段；响应字段为 session_id、task_id、agent_id、action。详见 docs/API_CHANGELOG.md §1。
// 请求
POST /api/tasks
Content-Type: application/json

// 基础对话（Phase 1+）
{
    "action": "chat",
    "agent_id": "agent_research_001",
    "input": "帮我分析最新的 AI 发展趋势"
}

// 流式 Demo（Phase 0 测试用）
{
    "action": "stream-demo"
}

// 预设 Case（Phase 3+）
{
    "action": "case",
    "case_id": "code_gen_and_test"
}

// 响应
HTTP 200
{
    "task_id": "task_20260703103000",
    "status": "started"
}

6.6 Tool 注册（Phase 5+）

运行时工具注册已实现，且请求体必须包含 type 字段（shell/http/inline）及对应子字段。详见 docs/API_CHANGELOG.md §2.3。
// 注册新工具
POST /api/tools
{
    "name": "web_search",
    "description": "Search the web for information",
    "schema": {
        "type": "object",
        "properties": {
            "query": { "type": "string", "description": "Search query" }
        },
        "required": ["query"]
    }
}
