1. 产品概述

描述产品的目标、定位、核心价值和使用场景。

1.1 产品定位

Multi-Agent Platform 是一个可实时观察、可扩展、可配置的 AI Agent 协作平台。

它同时服务于两种角色：

  • 学习工具：作为研究 Agent 架构、Go 后端开发、Vue 实时可视化的实验场
  • 内部产品：作为团队内部使用的多 Agent 任务执行平台
核心理念
"看见 AI 如何思考" —— 不只是看结果，而是看整个思考-行动-观察的循环过程。

1.2 目标用户

用户类型特征核心需求
开发者（主要）Go / Agent 技术栈学习者实时观察 Agent 行为、理解 ReAct Loop、调试 Tool 调用
内部用户团队成员一键发起预设任务、查看执行结果、管理 Agent 配置

1.3 核心价值

价值点说明
🔍 全链路可视化 实时展示 Agent 的每一步：思考过程、工具调用、观察结果、token 消耗
⚡ 多 Agent 并行 支持多个 Agent 同时执行不同的子任务，前端实时合并状态树
🔧 高度可配置 每个 Agent 可独立配置 API endpoint、model、system prompt、允许的工具集
🧩 运行时扩展 工具支持运行时注册（API / AI 自描述），无需重启服务
💾 持久化一切 SQLite 存储所有配置、执行历史、对话记录、文件元数据
🛠️ 学习优先 自研 Agent Loop（不封装 langchaingo），最大化学习价值

1.4 系统边界

范围内 ✅范围外 ❌
  • 单仓库 monorepo（Go + Vue）
  • 内部 LLM API（deepseek-v4-flash）
  • 本地 SQLite 持久化
  • WebSocket 实时通信
  • 3 个内置工具（shell / file ops）
  • 5 个预设任务场景
  • 外部 LLM Provider（仅内部 API）
  • 用户认证 / 多租户（Phase 6+）
  • RAG 向量检索（Phase 6+）
  • gRPC（预留接口，Phase 0-4 用 WS）
  • Docker 沙箱（Phase 5+）
  • 移动端 UI

1.5 设计原则

原则解释
渐进式构建 分 Phase 实现，每个 Phase 独立可验证，不追求一步到位
可见性优先 Agent 内部状态尽可能暴露给前端，支持选择性隐藏
协议可替换 后端 EventBus 接口抽象，WebSocket ↔ gRPC 可无缝切换
数据不丢失 所有执行历史持久化到 SQLite，支持回放和审计
学习驱动 自研核心组件（Agent Loop / Tool Registry），而非封装第三方库
